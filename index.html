RANK A 진행 중 (Phase 2)

미니미 — 데스크탑 AI 컴패니언 (desktop-pet)

바탕화면 위를 돌아다니는 픽셀 캐릭터와 대화하는 상주형 데스크탑 AI 도구

무엇을 투명·항상 위·클릭통과 오버레이 위에서 캐릭터가 화면 하단을 배회하고, 더블클릭하면 채팅창이 열려 AI와 대화(스트리밍 응답, 페르소나, 세션 메모리)할 수 있는 Electron 앱.
왜 "항상 곁에 있는 AI"를 브라우저 탭이 아닌 OS 레벨에서 구현해 보고 싶었고, 창 플래그·IPC·프로세스 분리 같은 데스크탑 앱의 시스템 계층을 직접 다뤄 보기 위해.
어떻게 API 키·네트워크를 메인 프로세스에만 격리하고 렌더러에는 contextBridge IPC만 노출해 보안 경계를 지켰습니다. AI는 Provider 추상화로 설계해 Gemini·OpenAI 호환(LM Studio 포함) SSE 스트리밍 어댑터를 baseUrl 주입만으로 재사용합니다.

RANK A CLEAR (M2)

RP 채팅 프론트엔드 (rp-frontend)

실리태번 계열을 참고해 밑바닥부터 만든 AI 롤플레잉 채팅 웹앱

무엇을 캐릭터 카드·프리셋·세션 관리·백업/공유 기능을 갖춘 React SPA. Cloudflare Workers 프록시를 통해 LLM API에 연결하고, 모든 데이터는 브라우저 IndexedDB에 저장됩니다.
왜 기존 도구(실리태번)는 무겁고 설치가 필요해서, "브라우저만 있으면 되는" 가벼운 대안을 직접 설계·구현하며 프론트엔드 아키텍처와 API 연동 전 과정을 경험하기 위해.
어떻게 API 어댑터(buildRequest/parseResponse)를 앱 코드와 격리해 OpenAI·Gemini 등 어떤 응답 포맷이든 두 함수 수정만으로 대응합니다. 영속성은 Dexie(IndexedDB)로 서버 없이 해결하고, CSS 변수 기반 디자인 토큰으로 테마 시스템을 만들었습니다.
ReactVite IndexedDB (Dexie)Cloudflare Workers LLM API 연동

RANK B CLEAR

SQL 던전 — SQL 학습 게임 (sql game_final)

SQL 쿼리를 입력해 스테이지를 클리어하며 배우는 학습용 게임

무엇을 SELECT부터 JOIN·GROUP BY까지, 단계별 퀘스트로 구성된 문제를 사용자가 직접 SQL로 풀면 실제 데이터베이스에서 실행·채점되는 학습 게임.
왜 SQL은 "직접 쳐 봐야" 는다는 경험에서 출발했습니다. 눈으로 읽는 강의 대신 즉각적인 성공/실패 피드백이 있는 게임 형식이 학습 동기를 유지시켜 준다고 판단했습니다.
어떻게 Python에서 MariaDB에 직접 연결해 사용자의 쿼리를 실행하고, 정답 쿼리의 결과 집합과 비교하는 방식으로 채점해 다양한 풀이를 허용했습니다. 스테이지·힌트 데이터도 DB 테이블로 관리해 문제 추가가 코드 수정 없이 가능합니다.
